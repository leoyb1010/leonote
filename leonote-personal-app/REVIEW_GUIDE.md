# REVIEW GUIDE

## 给 Reviewer / Claude 的阅读顺序

建议按以下顺序 review：

1. `README.md`
2. `CHANGELOG.md`
3. `PRD.md`
4. `ARCHITECTURE.md`
5. `prisma/schema.prisma`
6. `src/app/api/**`
7. `src/components/**`

---

## 当前 review 重点

### 1. 安全
请重点看：
- session 是否足够稳妥
- 是否还存在公开初始化/后门式逻辑
- API 是否正确校验用户归属

### 2. 数据一致性
请重点看：
- Note / Tag / DailyNote / Project 之间关系是否合理
- 导入导出协议是否一致
- 项目归属逻辑是否清晰
- JSON 导入事务是否覆盖所有批量写入路径
- 数据库索引是否匹配当前列表/搜索/项目查询

### 3. 编辑器交互
请重点看：
- 自动保存开关逻辑是否合理
- 手动保存与自动保存是否还有冲突
- 保存成功弹窗是否合适
- 中文输入法 composition 期间是否仍可能误触发保存

### 4. 产品结构
请重点看：
- 项目是否适合作为一级入口
- 每日 / 项目 / 普通笔记三条线是否清晰
- 导入能力是否该继续扩展为附件系统

### 5. 工程可维护性
请重点看：
- API 与前端状态收口是否干净
- 是否存在明显重复逻辑
- 当前目录结构是否适合继续扩展

---

## 当前已知限制

- 当前导入白名单为 JSON / Markdown / TXT / HTML / 网页链接
- DOCX / PDF 暂不支持，后续应作为附件系统或专门解析链路评估
- 附件系统还未实现
- 项目模块目前是第一版，不是完整项目管理系统
- 当前偏单人使用，不考虑多人协作
- 当前还没有独立自动化测试套件，交付主要依赖 lint / typecheck / build

---

## 希望 reviewer 回答的问题

1. 当前数据结构是否适合继续迭代？
2. 自动保存开关的方案是否合理？
3. 项目模块当前抽象是否合适？
4. 导入能力的下一步应该优先做什么？
5. 代码里是否还有高风险点或明显结构问题？
6. 当前 migration baseline 是否满足部署和后续迁移需要？
